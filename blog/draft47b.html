_by / _by_key

Sort these log entries by timestamp. Reverse the order — Solution

March 2026 · Part B: The Solution

The answer: A

Use sort_by_key with Reverse.

sort_by_key extracts a comparable value from each element and sorts by it. The _by_key suffix means "compare by this extracted field." Wrapping the key in std::cmp::Reverse flips the ordering — descending sort in one expression.

The _by / _by_key naming convention appears across the standard library: sort_by, sort_by_key, min_by, min_by_key, max_by, max_by_key. Use _by_key when comparing a single property. Use _by when you need a full custom comparator.

The solution

use std::cmp::Reverse;

struct LogEntry {
    timestamp: u64,
    message: String,
}

fn main() {
    let mut logs = vec![
        LogEntry { timestamp: 1000, message: "start".into() },
        LogEntry { timestamp: 3000, message: "end".into() },
        LogEntry { timestamp: 2000, message: "middle".into() },
    ];

    logs.sort_by_key(|entry| Reverse(entry.timestamp));

    for log in &logs {
        println!("{}: {}", log.timestamp, log.message);
    }
    // 3000: end
    // 2000: middle
    // 1000: start
}

Why the other options fail

B. sort_by_key then .reverse() — sort_by_key(|e| e.timestamp) sorts ascending. Then .reverse() flips the vec. It works but requires two passes over the data. Reverse achieves descending order during the sort itself — one pass, and the intent is clear from the sort key.

C. sort_by with swapped a and b — sort_by(|a, b| b.timestamp.cmp(&a.timestamp)) works and is a common pattern. But for a single-field comparison, sort_by_key is simpler. You point at the field; the adapter handles the comparison. Save sort_by for multi-field tie-breaking or truly custom ordering logic.

D. sort() — sort() requires LogEntry: Ord, which isn't implemented. Custom types don't have natural ordering by default. And even if you derived Ord, it would compare all fields in declaration order — you couldn't choose timestamp-only or descending. _by_key gives you precise control.

_by_key for single-field comparisons. _by for custom logic. Wrap in Reverse for descending order.
