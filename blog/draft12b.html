This counter needs to be visible to three threads — Solution

March 2026 · Part B: The Solution

The answer: B

Wrap it in Arc<Mutex<HitCounter>>.

Two problems need solving here. First, multiple threads need to own the counter — you can't just hand a reference to a spawned thread because the thread might outlive the scope that created the counter. Arc (atomic reference counting) gives you shared ownership across threads. Each thread gets its own Arc handle, and the counter lives until the last handle is dropped.

Second, the counter needs mutation. Multiple threads can't all hold &mut at once — that's a data race, and Rust won't allow it. Mutex provides interior mutability with runtime exclusion: one thread locks, increments, unlocks. The others wait their turn.

Arc solves the ownership problem. Mutex solves the mutation problem. You almost always need both when sharing mutable state across threads.

The solution

use std::sync::{Arc, Mutex};
use std::thread;

struct HitCounter {
    count: u64,
}

impl HitCounter {
    fn new() -> Self {
        HitCounter { count: 0 }
    }

    fn increment(&mut self) {
        self.count += 1;
    }

    fn get(&self) -> u64 {
        self.count
    }
}

fn main() {
    let counter = Arc::new(Mutex::new(HitCounter::new()));
    let mut handles = vec![];

    for _ in 0..3 {
        let counter = Arc::clone(&counter);
        handles.push(thread::spawn(move || {
            for _ in 0..100 {
                counter.lock().unwrap().increment();
            }
        }));
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("hits: {}", counter.lock().unwrap().get());
}

Why the other options fail

A. Rc<RefCell<>> — Rc is not Send. The compiler will refuse to move it into a spawned thread. Rc uses non-atomic reference counting, which would be a data race if shared across threads.

C. Mutex without Arc — you can't pass a reference to a Mutex into a spawned thread because the thread might outlive the Mutex. Spawned threads require 'static data unless you use scoped threads.

D. static mut — every access to static mut requires unsafe, and concurrent mutation without synchronisation is undefined behaviour. This is exactly the kind of bug Rust exists to prevent.

Arc for sharing across threads, Mutex for mutating shared data — they're a pair.
