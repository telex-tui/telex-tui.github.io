This config loads from disk. Load it once, share it everywhere — Solution

March 2026 · Part B: The Solution

The answer: A

Use OnceLock with a static variable.

OnceLock is the standard library's thread-safe once-cell for statics. You declare a static OnceLock<AppConfig>, initialize it once at startup, and every subsequent access returns &'static AppConfig — a reference that lives for the entire program.

This is the idiomatic Rust approach for read-only global state. The config is immutable after initialization, so there's no synchronization overhead after the first access. Any function anywhere in the program can call config() and get a reference.

The alternative — passing &Config through every function — is cleaner for testing and smaller programs. But for application-level config that genuinely needs to be everywhere, a OnceLock static is pragmatic and safe.

The solution

use std::sync::OnceLock;

struct AppConfig {
    db_host: String,
    db_port: u16,
    log_level: String,
}

impl AppConfig {
    fn load() -> Self {
        AppConfig {
            db_host: "localhost".into(),
            db_port: 5432,
            log_level: "info".into(),
        }
    }
}

static CONFIG: OnceLock<AppConfig> = OnceLock::new();

fn config() -> &'static AppConfig {
    CONFIG.get_or_init(AppConfig::load)
}

fn start_db() {
    let c = config();
    println!("connecting to {}:{}", c.db_host, c.db_port);
}

fn setup_logging() {
    let c = config();
    println!("log level: {}", c.log_level);
}

fn main() {
    start_db();
    setup_logging();
}

Why the other options fail

B. static mut — requires unsafe for every read and write. Multiple threads accessing a static mut is undefined behaviour. Even single-threaded, it's an unsafe footgun when OnceLock does the same thing safely.

C. thread_local! — each thread gets its own copy of the config. Changes in one thread aren't visible in another. For config that should be the same everywhere, this is wrong — it fragments state instead of sharing it.

D. Load every time — reads from disk on every access. Slow, and if the file changes between reads, different parts of the program see different configs. The whole point is to load once and share one consistent snapshot.

OnceLock for statics: initialise once, share a &'static reference everywhere.
