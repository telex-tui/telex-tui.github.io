_map combiners

Find the first line in a file that parses as a valid port number — Solution

March 2026 · Part B: The Solution

The answer: A

Use find_map.

find_map combines find and map: it scans an iterator for the first element where the closure returns Some(value), then returns that value. It short-circuits — no further elements are processed after the match. Exactly like the return Some(port) in the manual loop.

The _map suffix means "combined with a transformation." filter_map keeps all matches. find_map keeps only the first. Same naming convention, different scope.

The solution

fn first_valid_port(lines: &[&str]) -> Option<u16> {
    lines.iter().find_map(|line| {
        let port: u16 = line.trim().parse().ok()?;
        if port > 0 { Some(port) } else { None }
    })
}

fn main() {
    let config = vec!["# comment", "", "  8080  ", "443"];
    println!("{:?}", first_valid_port(&config));  // Some(8080)
}

The ? operator works inside the closure because it returns Option. If parse() fails, ok()? returns None from the closure, and find_map moves to the next element.

Why the other options fail

B. filter then map then next — three steps for what find_map does in one. And filter can't transform values — you'd need to parse in filter (to check), then parse again in map (to extract). Same double-parse problem as the original code with filter_map.

C. map then flatten then next — map produces Option<u16> for each line. flatten unwraps the Somes and discards the Nones. next takes the first. This works but is three operations for what find_map expresses directly. The name find_map tells the reader: "find the first transformable element."

D. Keep the loop — iterators short-circuit perfectly well. find_map stops at the first Some, just like return stops the loop. The iterator version is shorter and its name — find_map — declares the intent that the loop body only implies.

filter_map for "all matches, transformed." find_map for "first match, transformed." The _map suffix always means: test and transform in one pass.
