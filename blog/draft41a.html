as_ / to_ / into_

You need the raw bytes of a String. Borrow or consume?

March 2026 · Part A: The Challenge

A checksum function that works on byte slices. The caller has a String and needs it afterwards.

fn checksum(data: &[u8]) -> u32 {
    data.iter().map(|&b| b as u32).sum()
}

fn main() {
    let payload = String::from("hello world");
    let bytes = payload.clone().into_bytes();
    println!("checksum: {}", checksum(&bytes));
    println!("original: {}", payload);
}

.clone().into_bytes() allocates a full copy of the String just to get its bytes. The original is still needed. Fix it.

Your options

  • Use payload.as_bytes() — borrows the bytes, zero allocation
  • Keep the clone — consuming is the only way to get bytes from a String
  • Use payload.as_bytes().to_vec() — owned bytes without consuming
  • Use payload.into_bytes() without the clone

Pick one before reading the solution.
