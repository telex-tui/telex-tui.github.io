as_ / to_ / into_

This API takes an owned String. You have a &str. Cheapest path

March 2026 · Part A: The Challenge

A network function that takes owned bytes. The caller has a String that's never used again.

fn send_packet(data: Vec<u8>) {
    println!("sending {} bytes", data.len());
}

fn main() {
    let message = String::from("PING");
    let packet = message.as_bytes().to_vec();
    send_packet(packet);
    // message is never used again
}

as_bytes() borrows the bytes, then to_vec() copies them into a new Vec. But message is never used after this. There's a zero-copy path. Find it.

Your options

  • Use message.into_bytes() — consumes the String, reuses its buffer
  • Keep as_bytes().to_vec() — the copy is needed for type safety
  • Use message.as_bytes() and change send_packet to take &[u8]
  • Use message.to_bytes() — to_ creates an owned copy

Pick one before reading the solution.
