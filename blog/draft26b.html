A cache shared across tasks. It needs to initialise exactly once — Solution

March 2026 · Part B: The Solution

The answer: A

Use tokio::sync::OnceCell.

OnceCell guarantees exactly-once initialization, even when multiple tasks call get_or_init concurrently. The first caller runs the init closure. All other callers wait for it to complete and then get the result. No double-initialization, no race conditions, no manual locking.

The tokio version (as opposed to std::sync::OnceLock) is specifically designed for async contexts. The init closure can be async — you could do I/O, make network calls, or read from disk during initialization, and it integrates cleanly with the async runtime.

After initialization, get_or_init returns a reference to the stored value with no locking overhead. It's effectively free after the first call.

The solution

use std::collections::HashMap;
use std::sync::Arc;
use tokio::sync::OnceCell;

fn build_lookup() -> HashMap<String, u64> {
    println!("building lookup (expensive)...");
    let mut map = HashMap::new();
    map.insert("alpha".into(), 1);
    map.insert("beta".into(), 2);
    map.insert("gamma".into(), 3);
    map
}

// Shared across tasks, initialised exactly once
static LOOKUP: OnceCell<HashMap<String, u64>> = OnceCell::const_new();

async fn get_lookup() -> &'static HashMap<String, u64> {
    LOOKUP.get_or_init(|| async { build_lookup() }).await
}

#[tokio::main]
async fn main() {
    let mut handles = vec![];

    for i in 0..3 {
        handles.push(tokio::spawn(async move {
            let table = get_lookup().await;
            println!("task {}: alpha = {:?}", i, table.get("alpha"));
        }));
    }

    for h in handles {
        h.await.unwrap();
    }
}

Why the other options fail

B. Mutex<Option<>> — requires holding the lock during the entire init, blocking all other tasks. And you have to write the check-and-init logic yourself, getting the locking scope right. OnceCell does this correctly by design.

C. lazy_static! — works for synchronous initialization, but the init closure can't be async. If building the lookup requires I/O, you're blocking the async runtime.

D. Build in each task — duplicates expensive work. If three tasks race to build it, you pay the cost three times and use three times the memory. The whole point is to build once.

For once-only async initialization shared across tasks, OnceCell is the right primitive.
