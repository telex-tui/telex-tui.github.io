Three types share behaviour. Trait or enum?

March 2026 · Part A: The Challenge

Three message types that all need to be formatted for a chat log.

struct TextMessage {
    sender: String,
    body: String,
}

struct ImageMessage {
    sender: String,
    url: String,
}

struct SystemMessage {
    text: String,
}

fn main() {
    let msg = TextMessage { sender: "Alice".into(), body: "hello".into() };
    println!("[{}] {}", msg.sender, msg.body);
}

Store a mixed list of these messages and iterate through them, formatting each one for display.

Your options

  • Use an enum with three variants — the set of message types is known and closed
  • Define a Message trait and store Vec<Box<dyn Message>>
  • Use generics with a trait bound to handle all three types uniformly
  • Always use traits because that's the proper object-oriented approach

Pick one before reading the solution.
