copied vs cloned

This iterator yields &u32. Make it yield u32 and signal the cost

March 2026 · Part A: The Challenge

A function that sums even numbers from a slice.

fn sum_even(numbers: &[u32]) -> u32 {
    numbers.iter()
        .filter(|&&n| n % 2 == 0)
        .cloned()
        .sum()
}

fn main() {
    let data = vec![1, 2, 3, 4, 5, 6];
    println!("sum of evens: {}", sum_even(&data));
}

The code uses .cloned() to convert &u32 to u32. It compiles and runs correctly. But the choice of adapter sends the wrong signal about cost. Fix it.

Your options

  • Use .copied() — signals a trivially cheap bitwise copy
  • Keep .cloned() — it works and is more general
  • Use .map(|&n| n) — manual destructuring avoids both
  • Remove it entirely — filter already produces owned values

Pick one before reading the solution.
