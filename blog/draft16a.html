The callback needs to outlive the thing that registered it

March 2026 · Part A: The Challenge

An event system that stores a callback and fires it later.

struct EventBus {
    listeners: Vec<fn()>,
}

impl EventBus {
    fn new() -> Self {
        EventBus { listeners: vec![] }
    }

    fn on_event(&mut self, callback: fn()) {
        self.listeners.push(callback);
    }

    fn fire(&self) {
        for listener in &self.listeners {
            listener();
        }
    }
}

fn main() {
    let mut bus = EventBus::new();
    bus.on_event(|| println!("event fired!"));
    bus.fire();
}

The callback currently can't capture any state — it's a bare function pointer. Make it support closures that capture variables from their environment.

Your options

  • Store closures as Box<dyn Fn()> and require them to own all their captured data
  • Store closures as &dyn Fn() with a lifetime on the EventBus
  • Keep using function pointers but pass captured state as a separate *mut () argument
  • Store closures as generic type parameters on EventBus

Pick one before reading the solution.
