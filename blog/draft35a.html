Fan out to three tasks. Collect all results. Handle one failure

March 2026 · Part A: The Challenge

Three independent data fetches that all need to complete.

async fn fetch_users() -> Result<Vec<String>, String> {
    Ok(vec!["Alice".into(), "Bob".into()])
}

async fn fetch_orders() -> Result<Vec<String>, String> {
    Ok(vec!["order-1".into(), "order-2".into()])
}

async fn fetch_products() -> Result<Vec<String>, String> {
    Ok(vec!["widget".into(), "gadget".into()])
}

#[tokio::main]
async fn main() {
    let users = fetch_users().await.unwrap();
    let orders = fetch_orders().await.unwrap();
    let products = fetch_products().await.unwrap();
    println!("users: {:?}, orders: {:?}, products: {:?}", users, orders, products);
}

Run all three fetches concurrently. If any one fails, return its error immediately without waiting for the others.

Your options

  • Use tokio::try_join! — runs all concurrently, short-circuits on the first error
  • Spawn three tasks and .await them one by one in sequence
  • Use tokio::join! and check for errors after all three complete
  • Use tokio::select! to return whichever finishes first

Pick one before reading the solution.
