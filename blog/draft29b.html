Three types share behaviour. Trait or enum? — Solution

March 2026 · Part B: The Solution

The answer: A

Use an enum. The set of message types is closed.

You know all three types at compile time. No one outside your codebase will add a fourth. This is exactly what Rust enums are designed for. An enum with three variants is simpler, faster, and more powerful than a trait hierarchy.

Simpler: no vtable indirection, no Box allocation, no trait object. The enum lives on the stack. Faster: pattern matching is a direct jump, not a virtual dispatch. More powerful: match is exhaustive — if you add a fourth variant later, the compiler tells you every place that needs updating. With a trait, you'd silently miss call sites.

Traits are for when you need extensibility — when other crates or modules will add new types that implement the same interface. If you control all the variants, use an enum.

The solution

enum Message {
    Text { sender: String, body: String },
    Image { sender: String, url: String },
    System { text: String },
}

impl Message {
    fn format_for_log(&self) -> String {
        match self {
            Message::Text { sender, body } => format!("[{}] {}", sender, body),
            Message::Image { sender, url } => format!("[{}] (image: {})", sender, url),
            Message::System { text } => format!("*** {} ***", text),
        }
    }
}

fn main() {
    let log: Vec<Message> = vec![
        Message::Text { sender: "Alice".into(), body: "hello".into() },
        Message::Image { sender: "Bob".into(), url: "cat.png".into() },
        Message::System { text: "Alice joined".into() },
    ];

    for msg in &log {
        println!("{}", msg.format_for_log());
    }
}

Why the other options fail

B. Trait with Box<dyn> — adds heap allocation for every message, virtual dispatch for every method call, and loses exhaustive checking. Use this when you need extensibility. You don't here.

C. Generics with trait bounds — generics are monomorphised: Vec<T> can only hold one concrete type. You can't put a TextMessage and an ImageMessage in the same Vec<T>. Generics solve "same operation on different types" but not "different types in the same collection."

D. "Always use traits" (OOP mindset) — Rust is not Java. Enums with pattern matching replace most uses of inheritance hierarchies. Reaching for traits by default is bringing OOP habits into a language that has better tools.

Closed set of types? Enum. Open set? Trait. Default to enum.
