_by / _by_key

Sort these log entries by timestamp. Reverse the order

March 2026 · Part A: The Challenge

Log entries need to be sorted newest-first by timestamp.

struct LogEntry {
    timestamp: u64,
    message: String,
}

fn main() {
    let mut logs = vec![
        LogEntry { timestamp: 1000, message: "start".into() },
        LogEntry { timestamp: 3000, message: "end".into() },
        LogEntry { timestamp: 2000, message: "middle".into() },
    ];

    logs.sort_by(|a, b| {
        if a.timestamp > b.timestamp {
            std::cmp::Ordering::Less
        } else if a.timestamp < b.timestamp {
            std::cmp::Ordering::Greater
        } else {
            std::cmp::Ordering::Equal
        }
    });

    for log in &logs {
        println!("{}: {}", log.timestamp, log.message);
    }
}

The manual ordering logic in sort_by is verbose for a single-field comparison. There's a simpler adapter for sorting by an extracted key, and a standard way to reverse the order.

Your options

  • Use sort_by_key with Reverse — extract the key, wrap in Reverse for descending
  • Use sort_by_key then call .reverse() on the vec
  • Use sort_by with b.timestamp.cmp(&a.timestamp) — swap a and b
  • Use sort() — natural ordering handles it

Pick one before reading the solution.
