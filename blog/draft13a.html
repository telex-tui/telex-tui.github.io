Make this cache lazy — only compute on first access

March 2026 · Part A: The Challenge

This config struct eagerly computes an expensive derived value at construction time.

use std::collections::HashMap;

struct Config {
    raw: HashMap<String, String>,
    summary: String, // expensive to compute
}

impl Config {
    fn new(raw: HashMap<String, String>) -> Self {
        let summary = Self::build_summary(&raw);
        Config { raw, summary }
    }

    fn build_summary(raw: &HashMap<String, String>) -> String {
        // Simulate expensive work
        raw.iter()
            .map(|(k, v)| format!("{}={}", k, v))
            .collect::<Vec<_>>()
            .join("; ")
    }

    fn summary(&self) -> &str {
        &self.summary
    }
}

fn main() {
    let mut data = HashMap::new();
    data.insert("host".into(), "localhost".into());
    data.insert("port".into(), "8080".into());
    let config = Config::new(data);
    println!("{}", config.summary());
}

Make the summary computation lazy — it should only run the first time summary() is called.

Your options

  • Use OnceCell<String> to store the summary, initializing it on first access with get_or_init
  • Use Option<String> with a boolean flag to track whether the summary has been computed
  • Use lazy_static! to declare the summary as a lazily-initialized global
  • Use RefCell<Option<String>> to allow mutation through a shared reference

Pick one before reading the solution.
