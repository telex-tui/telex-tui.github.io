_while

Read lines from a stream. Stop at the first blank line — Solution

March 2026 · Part B: The Solution

The answer: A

Use take_while.

take_while yields elements as long as the predicate returns true. The moment the predicate returns false, the iterator stops — permanently. No further elements are processed. It's the declarative version of a for loop with break.

The _while suffix transforms a count-based operation into a predicate-based one. take(3) takes exactly 3 items. take_while(pred) takes items while a condition holds. Same concept, dynamic stopping point.

The solution

fn main() {
    let lines = vec!["Host: localhost", "Port: 8080", "Accept: */*", "", "body here", "more body"];

    let headers: Vec<&str> = lines.iter()
        .copied()
        .take_while(|line| !line.is_empty())  // stop at first blank line
        .collect();

    println!("{:?}", headers);
    // ["Host: localhost", "Port: 8080", "Accept: */*"]
}

The blank line and everything after it are never processed. take_while short-circuits just like break in the loop.

Why the other options fail

B. filter — filter removes blank lines but doesn't stop. It would skip the blank line and continue, including "body here" and "more body" in the results. filter is for "remove matching elements." take_while is for "stop at the boundary."

C. take(n) — take(n) requires knowing the count upfront. Headers can have any number of lines. The stopping point is a condition (blank line), not a count. That's exactly what _while is for: replacing a fixed count with a dynamic predicate.

D. skip_while — skip_while skips elements while the predicate is true, then yields everything after. skip_while(|l| !l.is_empty()) would skip the headers and yield from the blank line onward — the exact opposite of what you want. take_while takes the prefix. skip_while takes the suffix.

_while turns counts into conditions. take_while = take until false. skip_while = skip until false. Both short-circuit.
