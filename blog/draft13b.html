Make this cache lazy — only compute on first access — Solution

March 2026 · Part B: The Solution

The answer: A

Use OnceCell<String> to store the summary.

OnceCell is purpose-built for this pattern: a value that starts empty and gets initialized exactly once. You call get_or_init with a closure, and it either returns the already-computed value or runs the closure, stores the result, and returns it. Every subsequent call just returns the cached value. No flags, no option-checking, no interior mutability gymnastics.

The key insight is that OnceCell gives you interior mutability behind a &self reference — the summary() method takes &self, not &mut self, even though it might write the cached value on first call. That's the whole point. Callers don't need to know or care whether the value is computed yet.

For thread-safe code, use OnceLock (the thread-safe version). For single-threaded code, OnceCell from std::cell is enough.

The solution

use std::cell::OnceCell;
use std::collections::HashMap;

struct Config {
    raw: HashMap<String, String>,
    summary: OnceCell<String>,
}

impl Config {
    fn new(raw: HashMap<String, String>) -> Self {
        Config {
            raw,
            summary: OnceCell::new(), // starts empty
        }
    }

    fn build_summary(raw: &HashMap<String, String>) -> String {
        raw.iter()
            .map(|(k, v)| format!("{}={}", k, v))
            .collect::<Vec<_>>()
            .join("; ")
    }

    fn summary(&self) -> &str {
        // Computed on first call, cached for all subsequent calls
        self.summary.get_or_init(|| Self::build_summary(&self.raw))
    }
}

fn main() {
    let mut data = HashMap::new();
    data.insert("host".into(), "localhost".into());
    data.insert("port".into(), "8080".into());
    let config = Config::new(data);
    // summary is only computed here, on first access
    println!("{}", config.summary());
    println!("{}", config.summary()); // second call uses cached value
}

Why the other options fail

B. Option with boolean flag — works, but you're reinventing what OnceCell already does. You also need &mut self to set the Option, which means your summary() method can't take &self without additional interior mutability.

C. lazy_static! — that's for global statics, not struct fields. Your summary depends on instance data (the specific raw map), so a global doesn't work.

D. RefCell<Option<>> — works but is more complex than necessary. You'd need to borrow, check, and set manually. OnceCell wraps this exact pattern with a clean API and zero overhead after initialization.

OnceCell is the standard tool for lazy, compute-once fields.
