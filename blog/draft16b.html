The callback needs to outlive the thing that registered it — Solution

March 2026 · Part B: The Solution

The answer: A

Store closures as Box<dyn Fn()>.

A closure that captures variables by reference borrows from the scope where it was created. If the EventBus outlives that scope, those references dangle. The fix: make the closure own its captures. A move closure captures by value, and Box<dyn Fn()> puts it on the heap so the EventBus can store it without knowing its concrete type or size.

The dyn Fn() trait object erases the closure's type. Every closure in Rust is a unique anonymous type — you can't store a Vec of different closures without type erasure. Box<dyn Fn()> is the standard way to store a heterogeneous collection of callables.

Adding a 'static bound (which is implicit on Box<dyn Fn()>) ensures every closure owns all its data. No dangling references, no lifetime headaches.

The solution

struct EventBus {
    // Box<dyn Fn()> stores any closure that takes no args and returns nothing.
    // The implicit 'static bound means closures must own their captures.
    listeners: Vec<Box<dyn Fn()>>,
}

impl EventBus {
    fn new() -> Self {
        EventBus { listeners: vec![] }
    }

    fn on_event(&mut self, callback: impl Fn() + 'static) {
        self.listeners.push(Box::new(callback));
    }

    fn fire(&self) {
        for listener in &self.listeners {
            listener();
        }
    }
}

fn main() {
    let mut bus = EventBus::new();

    let name = String::from("Alice");
    // move captures `name` by value — the closure owns it
    bus.on_event(move || println!("Hello, {}!", name));
    bus.on_event(|| println!("anonymous event"));

    bus.fire();
}

Why the other options fail

B. &dyn Fn() with a lifetime — ties every callback's lifetime to the EventBus, which means the EventBus can't outlive any callback's source scope. This makes the event bus nearly useless in real code.

C. Function pointers with *mut () — this is the C callback pattern. It works but requires unsafe everywhere, throws away type safety, and makes you manually manage the state pointer's lifetime. Rust has closures for a reason.

D. Generic type parameter — a generic EventBus<F: Fn()> can only store one concrete closure type. You can't put two different closures in the same Vec because they have different types. That's why you need dyn — type erasure.

Box<dyn Fn()> is how you store callbacks that outlive their origin: heap-allocated, type-erased, and owning their data.
