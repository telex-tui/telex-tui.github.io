_map combiners

You have Option<Option<T>>. Flatten it without matching

March 2026 · Part A: The Challenge

A nested lookup that returns Option<Option<i32>>. The caller needs a flat Option<i32>.

fn lookup(key: &str) -> Option<Option<i32>> {
    match key {
        "a" => Some(Some(1)),
        "b" => Some(None),
        _ => None,
    }
}

fn main() {
    let value = match lookup("a") {
        Some(Some(v)) => Some(v),
        Some(None) => None,
        None => None,
    };
    println!("{:?}", value);
}

The nested match manually collapses two layers of Option. There's a one-method solution. Use it.

Your options

  • Use .flatten() — collapses Option<Option<T>> into Option<T>
  • Use .and_then(|x| x) — and_then is Option's flat_map
  • Use .map(|x| x.unwrap_or_default()) — unwrap the inner Option
  • Use .unwrap().unwrap() — double unwrap handles both layers

Pick one before reading the solution.
