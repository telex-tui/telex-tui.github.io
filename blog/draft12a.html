This counter needs to be visible to three threads

March 2026 · Part A: The Challenge

A simple hit counter. Works fine on one thread.

struct HitCounter {
    count: u64,
}

impl HitCounter {
    fn new() -> Self {
        HitCounter { count: 0 }
    }

    fn increment(&mut self) {
        self.count += 1;
    }

    fn get(&self) -> u64 {
        self.count
    }
}

fn main() {
    let mut counter = HitCounter::new();
    counter.increment();
    counter.increment();
    println!("hits: {}", counter.get());
}

Make this counter usable from three separate threads that all increment it concurrently.

Your options

  • Wrap it in Rc<RefCell<HitCounter>> and share the Rc across threads
  • Wrap it in Arc<Mutex<HitCounter>> and clone the Arc for each thread
  • Use a Mutex<HitCounter> without Arc, passing a reference to each thread
  • Declare it as static mut so all threads can access it directly

Pick one before reading the solution.
