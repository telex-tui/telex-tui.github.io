Add logging to this without changing every call site

March 2026 · Part A: The Challenge

A simple HTTP client used across multiple functions. None of them have a logger.

struct Client {
    base_url: String,
}

impl Client {
    fn new(base_url: &str) -> Self {
        Client { base_url: base_url.into() }
    }

    fn get(&self, path: &str) -> String {
        format!("GET {}{}", self.base_url, path)
    }
}

fn fetch_users(client: &Client) -> String {
    client.get("/users")
}

fn fetch_orders(client: &Client) -> String {
    client.get("/orders")
}

fn main() {
    let client = Client::new("https://api.example.com");
    println!("{}", fetch_users(&client));
    println!("{}", fetch_orders(&client));
}

Add logging so every get() call is logged, without adding a logger parameter to fetch_users or fetch_orders.

Your options

  • Add a &Logger parameter to every function that uses the client
  • Put the logger inside the Client struct so logging happens automatically in get()
  • Use a global static mut logger that any function can access
  • Define a logging trait with a default method and have Client implement it

Pick one before reading the solution.
