_map combiners

You have Option<Option<T>>. Flatten it without matching — Solution

March 2026 · Part B: The Solution

The answer: A

Use .flatten().

Option::flatten() collapses Option<Option<T>> into Option<T>. Some(Some(v)) becomes Some(v). Both Some(None) and None become None. One method call replaces the entire nested match.

This connects to the _map combiner pattern: on iterators, flat_map = map + flatten. On Option, and_then = map + flatten. When you already have the nested structure and just need to collapse it, skip the map — call flatten directly.

The solution

fn lookup(key: &str) -> Option<Option<i32>> {
    match key {
        "a" => Some(Some(1)),
        "b" => Some(None),
        _ => None,
    }
}

fn main() {
    let value = lookup("a").flatten();
    println!("{:?}", value);  // Some(1)

    let missing = lookup("b").flatten();
    println!("{:?}", missing);  // None

    let unknown = lookup("z").flatten();
    println!("{:?}", unknown);  // None
}

Why the other options fail

B. and_then(|x| x) — this works. and_then maps the inner value and flattens, so .and_then(|x| x) is equivalent to .flatten(). But it's the long way around. flatten() exists precisely for this case. Use and_then when you need to transform the inner value, not when you just need to collapse a layer.

C. map(|x| x.unwrap_or_default()) — this changes the semantics. Some(None) becomes Some(0) instead of None. "Key exists but has no value" and "key doesn't exist" should both be None, not Some(0). unwrap_or_default invents a value where there shouldn't be one.

D. unwrap().unwrap() — panics. lookup("b") returns Some(None). First unwrap() gives None. Second unwrap() panics. lookup("z") returns None. First unwrap() panics. The nested match exists specifically to handle these cases safely.

flatten collapses nested wrappers. On iterators: Vec<Vec<T>> → Vec<T>. On Option: Option<Option<T>> → Option<T>. Same concept, same name.
