Two async tasks need to coordinate on a single result

March 2026 · Part A: The Challenge

A producer task computes a value. A consumer task needs to wait for it.

use std::sync::{Arc, Mutex};

fn compute_answer() -> u64 {
    // Expensive computation
    42
}

fn main() {
    let result = Arc::new(Mutex::new(None::<u64>));

    // Producer
    {
        let r = Arc::clone(&result);
        *r.lock().unwrap() = Some(compute_answer());
    }

    // Consumer
    let answer = result.lock().unwrap().unwrap();
    println!("answer: {}", answer);
}

Make this work with async tasks — the consumer should await the producer's result without busy-waiting or polling.

Your options

  • Use tokio::sync::oneshot — the producer sends once, the consumer awaits the receiver
  • Use tokio::sync::mpsc — multi-producer, single-consumer channel
  • Use Arc<Mutex<Option<T>>> with a loop that checks every 10ms
  • Use a std::sync::Condvar to notify the consumer when the result is ready

Pick one before reading the solution.
