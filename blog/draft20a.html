This function returns a reference. Why won't it compile?

March 2026 · Part A: The Challenge

A function that formats a user's display name.

struct User {
    first: String,
    last: String,
}

impl User {
    fn new(first: &str, last: &str) -> Self {
        User { first: first.into(), last: last.into() }
    }
}

fn display_name(user: &User) -> String {
    format!("{} {}", user.first, user.last)
}

fn main() {
    let user = User::new("Alice", "Smith");
    println!("{}", display_name(&user));
}

You want display_name to return &str instead of String to avoid allocation. But format! creates a new String — you can't return a reference to a local. Fix this.

Your options

  • Return an owned String — you can't return a reference to something the function just created
  • Add a lifetime parameter and return &'a str tied to the input user
  • Use Box::leak to turn the String into a &'static str
  • Store the formatted name as a field on User and return a reference to that

Pick one before reading the solution.
