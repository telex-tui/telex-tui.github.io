Two functions need the same string. Who owns it?

March 2026 · Part A: The Challenge

Here's a small program that builds a greeting and sends it somewhere.

fn main() {
    let name = String::from("Alice");
    let greeting = format!("Hello, {}!", name);

    log_message(greeting);
    send_to_client(greeting);
}

fn log_message(msg: String) {
    println!("[LOG] {}", msg);
}

fn send_to_client(msg: String) {
    println!("[SEND] {}", msg);
}

This doesn't compile — greeting is moved into log_message and then used again. Fix it so both functions can use the greeting.

Your options

  • Clone the string before passing it to the first function, so each function gets its own copy
  • Change both functions to accept &str and pass references instead of owned strings
  • Wrap the string in Rc<String> so both functions share ownership via reference counting
  • Pass the string to the first function and have it return the string back so you can pass it to the second

Pick one before reading the solution.
