You clone everywhere to make it compile. What's the real fix?

March 2026 · Part A: The Challenge

A program that finds the longest item in a list, then adds to the list. Cloning is used to dodge the borrow checker.

fn main() {
    let mut items = vec![
        String::from("apple"),
        String::from("banana"),
        String::from("cherry"),
    ];

    // Clone to avoid borrow conflict
    let longest = items.iter()
        .max_by_key(|s| s.len())
        .cloned()
        .unwrap();

    items.push(format!("best: {}", longest));

    for item in &items {
        println!("{}", item);
    }
}

Remove the .cloned() call. Make this work without cloning.

Your options

  • Scope the immutable borrow so it ends before the mutable borrow begins — extract the needed data before mutating
  • Use Rc<String> for each element so the "longest" reference is shared, not cloned
  • Use Cow<str> for all strings so borrowing and owning are interchangeable
  • Accept the clone — this is just how Rust works when you need to read and write the same collection

Pick one before reading the solution.
