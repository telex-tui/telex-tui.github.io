This value needs to live longer than the function that created it

March 2026 · Part A: The Challenge

A function that creates a config and spawns a thread to use it.

use std::thread;

struct DatabaseConfig {
    host: String,
    port: u16,
}

fn start_worker() {
    let config = DatabaseConfig {
        host: "localhost".into(),
        port: 5432,
    };

    println!("Config ready: {}:{}", config.host, config.port);
}

fn main() {
    start_worker();
}

Spawn a thread inside start_worker that uses config. The thread will outlive the function — make it work.

Your options

  • Use a move closure to transfer ownership of config into the thread
  • Annotate config with 'static lifetime so it lives forever
  • Use Box::leak to make the config permanently live on the heap
  • Wrap in Arc so the config is shared between the function and the thread

Pick one before reading the solution.
