This tree needs an iterator. Write one

March 2026 · Part A: The Challenge

An expression tree with integer values at the leaves.

enum Expr {
    Num(i64),
    Add(Box<Expr>, Box<Expr>),
    Mul(Box<Expr>, Box<Expr>),
}

fn eval(expr: &Expr) -> i64 {
    match expr {
        Expr::Num(n) => *n,
        Expr::Add(a, b) => eval(a) + eval(b),
        Expr::Mul(a, b) => eval(a) * eval(b),
    }
}

fn main() {
    // (2 + 3) * 4
    let tree = Expr::Mul(
        Box::new(Expr::Add(
            Box::new(Expr::Num(2)),
            Box::new(Expr::Num(3)),
        )),
        Box::new(Expr::Num(4)),
    );
    println!("result: {}", eval(&tree));
}

Write an iterator that yields all the leaf values (the Num variants) in depth-first order.

Your options

  • Use an explicit stack of &Expr references in an iterator struct with a lifetime parameter
  • Collect all values into a Vec<i64> first, then return the vec's iterator
  • Use recursion with yield to generate values lazily
  • Implement IntoIterator that consumes the tree, moving nodes out during iteration

Pick one before reading the solution.
